# find_references 仕様

シンボルの参照箇所を検索するツール。定義位置から全ての使用箇所を特定する。

## 動作

1. 指定位置のシンボル名を取得
2. `git grep` で候補ファイルを検索
3. 各候補を解析し、同一定義への参照を抽出

## パラメータ

| 名前 | 型 | 説明 |
|------|-----|------|
| file_path | string | 定義を含むファイルパス |
| line | number | 行番号（1始まり） |
| column | number | 列番号（1始まり） |

## 制限事項

### 同一パッケージ内マッチング

インターフェース経由の参照も検出するため、同一パッケージ内の同名シンボルは参照としてカウントされる。

### 検出できないケース

- ダックタイピング（明示的な `implements` がない構造的互換）
- プロジェクトスコープ外の参照（`git grep` で補完）

### パフォーマンス

一般的な名前（`execute`, `get`, `set` 等）は多数のファイルがヒットし、処理時間が増加する。
